# 작업 목록: [기능 이름]

**입력**: `docs/specs/[###-기능-이름]/` 에서 가져온 설계 문서
**전제조건**: plan.md (필수), spec.md (사용자 스토리 필수), research.md, data-model.md, contracts/

**테스트**: 아래 예시에는 테스트 작업이 포함되어 있습니다. 테스트는 선택사항이며, 기능 명세에서 명시적으로 요청된 경우에만 포함합니다.

**조직**: 작업은 사용자 스토리별로 그룹화되어 각 스토리를 독립적으로 구현하고 테스트할 수 있습니다.

## 형식: `[ID] [P?] [스토리] 설명`

- **[P]**: 병렬 실행 가능 (다른 파일, 의존성 없음)
- **[스토리]**: 이 작업이 속한 사용자 스토리 (예: US1, US2, US3)
- 설명에 정확한 파일 경로 포함

## 경로 관례

- **단일 프로젝트**: 저장소 루트에 `src/`, `tests/`
- **웹 앱**: `backend/src/`, `frontend/src/`
- **모바일**: `api/src/`, `ios/src/` 또는 `android/src/`
- 아래 경로는 단일 프로젝트를 가정하며, plan.md 구조에 따라 조정

## 의존성 및 실행 순서

### Phase 의존성

- **설정 (Phase 1)**: 의존성 없음 - 즉시 시작 가능
- **기반 (Phase 2)**: 설정 완료에 의존 - 모든 사용자 스토리 블로킹
- **사용자 스토리 (Phase 3+)**: 기반 단계 완료에 의존
  - 사용자 스토리는 병렬로 진행 가능 (인력 충분 시)
  - 또는 우선순위 순으로 순차 진행 (P1 → P2 → P3)
- **다듬기 (최종 Phase)**: 모든 원하는 사용자 스토리 완료에 의존

### 사용자 스토리 의존성

- **사용자 스토리 1 (P1)**: 기반 (Phase 2) 후 시작 가능 - 다른 스토리와 의존성 없음
- **사용자 스토리 2 (P2)**: 기반 (Phase 2) 후 시작 가능 - US1과 통합 가능하지만 독립적으로 테스트 가능해야 함
- **사용자 스토리 3 (P3)**: 기반 (Phase 2) 후 시작 가능 - US1/US2와 통합 가능하지만 독립적으로 테스트 가능해야 함

### 각 사용자 스토리 내

- 테스트 (포함된 경우) 구현 전에 먼저 작성하고 실패해야 함
- 모델 → 서비스 → 엔드포인트 순서
- 핵심 구현 → 통합 순서
- 스토리 완료 후 다음 우선순위로 이동

### 병렬 기회

- 모든 설정 작업 [P] 마크 - 병렬 실행 가능
- 모든 기반 작업 [P] 마크 - Phase 2 내에서 병렬 실행 가능
- 기반 단계 완료 후 모든 사용자 스토리 병렬 시작 가능
- 사용자 스토리의 모든 테스트 [P] 마크 - 병렬 실행 가능
- 스토리 내 모델 [P] 마크 - 병렬 실행 가능
- 다른 사용자 스토리 - 다른 팀 멤버가 병렬로 작업 가능

## 구현 전략

### MVP 먼저 (사용자 스토리 1만)

1. Phase 1 완료: 설정
2. Phase 2 완료: 기반 (중요 - 모든 스토리 블로킹)
3. Phase 3 완료: 사용자 스토리 1
4. **정지 및 검증**: 사용자 스토리 1 독립적으로 테스트
5. 준비되면 배포/데모

### 점진적 전달

1. 설정 + 기반 완료 → 기반 준비 완료
2. 사용자 스토리 1 추가 → 독립적으로 테스트 → 배포/데모 (MVP!)
3. 사용자 스토리 2 추가 → 독립적으로 테스트 → 배포/데모
4. 사용자 스토리 3 추가 → 독립적으로 테스트 → 배포/데모
5. 각 스토리는 이전 스토리를 깨뜨리지 않고 가치 추가

## 참고

- [P] 작업 = 다른 파일, 의존성 없음
- [스토리] 라벨 = 작업과 특정 사용자 스토리 매핑
- 각 사용자 스토리는 독립적으로 완료 및 테스트 가능해야 함
- 구현 전에 테스트 실패 확인
- 각 작업 또는 논리적 그룹 후 커밋
- 체크포인트에서 정지하여 스토리 독립적으로 검증
- 모호한 작업, 동일 파일 충돌, 스토리 간 의존성 피하기
